# FashionOS Dashboard Architecture & Features Summary
## Comprehensive Organization Plan

**Version:** 1.0  
**Context:** Centralized Event Command & Lifecycle Management  
**Focus:** Organizing all dashboards beyond CRM  

This document outlines the complete set of dashboards required to manage a high-stakes fashion event, detailing screens, AI features, and data flows.

---

## 1. Dashboard Ecosystem Overview

The FashionOS dashboard is not a single screen but a **network of 6 interconnected command centers**.

| Dashboard | Core Function | Primary User |
| :--- | :--- | :--- |
| **1. Command Center (Home)** | High-level health, risks, and daily briefing. | Event Director / Lead Producer |
| **2. Project Overview** | Timeline, phases, critical path, and budget. | Project Manager |
| **3. Sponsor CRM** | Pipeline, contracts, activations, and ROI. | Sponsorship Director |
| **4. Tasks & Deliverables** | Granular execution, asset tracking, and approvals. | Production Team / Coordinators |
| **5. Venue Management** | Maps, logistics, load-in/out, and safety. | Logistics Manager |
| **6. Designer & Casting** | Collections, looks, models, and fittings. | Creative Director / Casting Director |

## 2. Detailed Dashboard Specifications

*The "Morning Coffee" View*

---

## 4. Technical Implementation Notes

*   **State Management:** Unified store (Zustand/Context) to prevent data drift between dashboards.
*   **AI Orchestrator:** A central service that routes requests to the correct Gemini model (Pro for reasoning, Flash for simple tasks).
*   **Real-time Sync:** Supabase Realtime for live updates across all screens.
